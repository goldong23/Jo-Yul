## 1. Business purpose

현대의 대학생들은 학업뿐만 아니라 대외활동, 아르바이트 등 이른바 '갓생'을 추구하며 매우 바쁜 일상을 보낸다. 특히 대학 생활에서 조별 모임이 차지하는 비중은 상당하며, 한 설문조사에 따르면 대학생들은 한 학기 평균 3.34개의 조별 모임에 참여하는 것으로 나타났다. 조별 과제의 부담이 워낙 크다 보니, 수강신청 시 조별 과제 여부를 고려한다는 학생이 62.8%에 달할 정도다. 이처럼 필수적인 모임이 다수 진행됨에도 불구하고, 개인마다 시간표와 일정이 달라 모두가 모일 수 있는 회의 시간을 정하는 과정부터 극심한 스트레스가 발생하고 있다.

또한, 모임 운영을 위해 개설되는 무분별한 메신저 단체방은 심각한 '정보 피로도'를 유발한다. 대학생들을 대상으로 한 조사에 따르면, 응답자의 71%가 단톡방 공지 내용의 가독성이 부족하다고 느꼈으며 공지방이 정보 전달보다는 오히려 학우들에게 피로도를 높이는 공간이 되고 있다고 지적했다. 특히 65%의 학생들은 시급한 개선책으로 필수 공지와 단순 홍보 메시지의 분리를 꼽았다. 단체 내에서 특정 세미나나 행사가 기획될 때마다 파생되는 새로운 단톡방, 혹은 전체 공지방에서 쏟아지는 본인과 무관한 알림들은 정작 중요한 정보의 유실을 초래한다. 게다가 모임 내 '무임승차(프리라이더)' 문제를 최악의 꼴불견으로 꼽은 학생이 70.7%에 이르는 만큼, 각자의 역할과 산출물을 투명하게 관리할 수 있는 시스템의 부재도 큰 문제다.

종합 모임 관리 프로그램 "Jo:YUl(조율)"은 이러한 대학생 단체 및 조모임의 고질적인 비효율과 소통 피로도를 해결하기 위해 기획되었다. 본 시스템은 단순한 연락망을 넘어, 모임의 생성부터 일정 맞춤, 소통, 산출물 관리까지 전 과정을 매끄럽게 '조율'하는 것을 핵심 비즈니스 목적으로 한다.

첫째, 시스템이 회원들의 개인 일정을 교차 분석하여 최적의 회의 시간을 자동 추천함으로써 일정 조율에 낭비되는 시간을 없앤다. 둘째, 새로운 이벤트나 행사가 있을 때 전체 알림과 투표를 통해 참여자를 확정하고, 해당 인원들만으로 구성된 독립적인 그룹 워크스페이스를 자동 생성한다. 이를 통해 운영진은 필수 인원에게만 타겟팅된 일괄 공지를 보낼 수 있어, 불필요한 알림 공해를 차단하고 정보 전달의 가독성을 대폭 높일 수 있다. 셋째, 그룹별 산출물과 업무 진행 상황을 한곳에서 통합 관리하여 프리라이더를 방지하고 공정한 참여를 유도한다.

결론적으로 "Jo:YUl"은 소모적인 연락과 알림 스트레스를 원천적으로 제거하여, 회원들이 단체 활동의 본질적인 목적에만 온전히 집중할 수 있는 스마트하고 효율적인 협업 환경을 제공할 것이다.

### 1). Register member & Log-in

| :--- | :--- |
| **Actor** | Member, Administrator |
| **Description** | 단체 회원은 학번, 이름, 연락처를 기입하여 회원가입을 한다. 등록된 사용자는 로그인을 통해 시스템의 기능(일정 입력, 과제 제출 등)을 사용할 수 있다. 관리자 역시 별도의 관리자 계정으로 로그인하여 전체 현황을 파악한다. |

## 4. Concept of operation

### 1). Register member & Log-in
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 인가된 단체 회원만 시스템을 이용하도록 제한함. |
| **Approach** | 학번 등을 ID로 사용하여 중복 가입을 방지한다. DB에 등록된 정보와 일치해야만 로그인이 승인되며, 프로그램 기능을 쓸 수 있다. |
| **Dynamics** | 시스템의 세부 기능을 이용하고자 할 때 접근한다. |
| **Goals** | 사용자 인증을 통한 보안 유지 및 개인화된 일정 관리 제공. |

### 2). Input & Recommend Schedule
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 조원 간의 일정 조율에 소모되는 시간을 최소화함. |
| **Approach** | 캘린더 형태의 UI를 제공하여 불가능한 시간을 블록 처리한다. 시스템은 교집합 연산을 통해 가능한 시간을 도출해 낸다. |
| **Dynamics** | 단체 활동 초반 조가 구성되고 정기 모임 시간을 확정해야 할 때 사용한다. |
| **Goals** | 모두가 동의할 수 있는 모임 시간을 시스템이 객관적으로 제시함. |

### 3). Upload Task
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 조별 프로젝트 산출물과 각 팀원의 기여도를 체계적으로 관리함. |
| **Approach** | 게시판 형식의 UI에서 파일 첨부 및 텍스트 입력을 지원한다. 제출된 과제는 서버에 저장되며, 담당자의 진행 상태 바가 갱신된다. |
| **Dynamics** | 주간 목표에 따른 개인 할 일을 완료했을 때 사용한다. |
| **Goals** | 무임승차를 방지하고 프로젝트 진행 상황의 투명성을 확보함. |

### 4). Manage Event & Notification
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 특정 행사의 참여 인원을 쉽게 모집하고, 참여자들끼리만 타겟팅하여 원활하게 소통하기 위함. |
| **Approach** | 이벤트 등록 시 전체 회원에게 알림이 발송됨. 투표 기능(참여/불참)을 제공하고, 마감 시 '참여' 인원만으로 이루어진 팀 워크스페이스를 자동 생성함. 이후 해당 워크스페이스 내에 작성되는 공지사항이나 게시물은 소속 그룹원에게만 일괄 알림으로 전달됨. |
| **Dynamics** | 전체 회원 중 일부만 참석하는 세미나, 친목 행사, 단기 프로젝트 등을 기획하고 인원을 모집할 때 사용한다. |
| **Goals** | 수요 조사부터 참여자 명단 및 그룹 생성까지의 과정을 자동화하고, 미참여자에게 불필요한 알림이 가는 것을 방지함. |

## 5. Problem statement

Jo:YUl의 Problem Statement는 아래와 같다.

1. 인터넷이 연결되어 있지 않으면 시스템에 접속하여 일정을 동기화하거나 과제를 업로드할 수 없다.
2. 실제 외부 메일 서버(SMTP)를 연동하지 않고 내부 앱 알림(시스템 알림)으로 우선 대체한다.
3. 데이터베이스 서버를 대규모로 구축하기 전이므로, 초기 프로토타입에서는 텍스트 및 저용량 파일 위주의 업로드만 지원하며 대용량 영상 파일 업로드는 제한한다.

## 6. Glossary

| Terms | Description |
| :--- | :--- |
| **조장/팀장** | 생성된 프로젝트 조의 리더 역할을 수행하며, 팀원 초대 및 일정 확정 권한을 가진다. |
| **산출물** | 스터디나 프로젝트를 통해 생성된 코드, 문서, 발표 자료 등을 포괄하여 이르는 말이다. |
| **블록 처리** | 캘린더 UI 상에서 본인이 참석 불가능한 시간대를 클릭 혹은 드래그하여 시스템에 불가 시간으로 인식시키는 행위이다. |
| **이벤트(Event)** | 전체 회원을 대상으로 투표를 진행하여, 참여 여부에 따라 별도의 소통 및 관리 그룹으로 자동 분리해야 하는 특정 행사나 단기 모임을 뜻한다. |
