## 1. Introduction

### 1) Executive Summary
현대의 대학생들은 학업뿐만 아니라 대외활동, 아르바이트 등 이른바 '갓생'을 추구하며 매우 바쁜 일상을 보낸다. 특히 대학 생활에서 조별 모임이 차지하는 비중은 상당하며, 개인마다 시간표와 일정이 달라 모두가 모일 수 있는 회의 시간을 정하는 과정부터 극심한 스트레스가 발생하고 있다. 또한, 모임 운영을 위해 개설되는 무분별한 메신저 단체방은 심각한 '정보 피로도'를 유발한다. 응답자의 71%가 단톡방 공지 내용의 가독성이 부족하다고 느꼈으며, 시급한 개선책으로 필수 공지와 단순 홍보 메시지의 분리를 꼽았다. 이러한 비효율과 피로도 문제를 해결하기 위해 종합 모임 관리 프로그램 "Jo:YUl(조율)"을 기획하였다.

### 2) Business Goals
본 시스템은 단순한 연락망을 넘어, 모임의 생성부터 일정 맞춤, 소통, 산출물 관리까지 전 과정을 매끄럽게 '조율'하는 것을 핵심 비즈니스 목적으로 한다. 
첫째, 시스템이 회원들의 개인 일정을 교차 분석하여 최적의 회의 시간을 자동 추천한다. 
둘째, 새로운 이벤트나 행사가 있을 때 전체 알림과 투표를 통해 참여자를 확정하고, 해당 인원들만으로 구성된 독립적인 그룹 워크스페이스를 자동 생성하여 필수 인원에게만 타겟팅된 일괄 공지를 보낸다. 
셋째, 그룹별 산출물과 업무 진행 상황을 한곳에서 통합 관리하여 무임승차(프리라이더)를 방지하고 공정한 참여를 유도한다.

### 3) Technical Goals
프로토타입 단계에서는 대규모 데이터베이스 서버 구축 전이므로, 실제 외부 메일 서버(SMTP)를 연동하지 않고 내부 앱(시스템) 알림으로 우선 대체하여 개발한다. 또한 파일 업로드의 경우 텍스트 및 저용량 파일 위주로 지원하며 대용량 영상 파일 업로드는 제한하여 서버 과부하를 방지한다. 오프라인 상태의 제한을 고려하여 인터넷 연결을 기반으로 데이터 및 일정을 동기화하는 구조로 설계한다.

## 3. Domain analysis

본 시스템(Jo:YUl)을 구성하는 주요 클래스 및 도메인 요소는 다음과 같다.
1) **Member**: 시스템을 이용하는 주체. 학번, 이름, 연락처 등의 속성을 가지며, 일반 회원과 관리자, 조장 등으로 나뉜다.
2) **Team**: 관리자에 의해 최초 생성되며 조원들의 묶음을 관리하는 도메인이다.
3) **Schedule**: 회원들의 불가 시간 데이터를 담고 있으며, 시스템이 이를 교차 연산하여 회의 시간을 추천하는 데 사용된다.
4) **Task/Mission**: 관리자가 공지한 주차별 할당 업무와 회원이 제출한 산출물, 그리고 이에 대한 점수 및 상태 바 데이터를 저장한다.
5) **Event**: 관리자가 주최하는 단기 모임이나 행사 도메인. 전체 대상 참여 투표 상태 및 알림 데이터를 가진다.
6) **Team Workspace**: 이벤트 투표 결과(참여자)를 기반으로 자동 생성되는 독립적인 소통 공간이다. 내부적으로 공지, 대화, 익명 등 세부 채널 속성을 갖는다.
7) **Server / System**: 데이터베이스를 동기화하고, 일정 추천, 워크스페이스 생성, 투표 미참여자 알림 발송 등 백그라운드 알고리즘을 수행하는 중추 역할을 한다.

## 4. User Interface prototype

### 4.1. Register & Log-in UI
모임 멤버나 관리자가 프로그램을 실행하면 학번과 비밀번호를 입력할 수 있는 직관적인 로그인 폼을 볼 수 있다. 중복 가입 방지를 위해 학번을 ID로 사용하는 회원가입 창으로 진입할 수 있는 버튼이 함께 배치되어 있다. 

### 4.2. Schedule Coordination UI
일정 기능을 클릭하면 주간/월간 캘린더 인터페이스가 나타난다. 본인이 불가능한 시간대를 마우스 드래그 혹은 클릭으로 손쉽게 '블록 처리(음영)' 할 수 있다. 회원들이 입력을 마치면 시스템이 교합하여 도출된 '전원 참석 가능 시간'이 캘린더 위에 하이라이트 표시되어 관리자가 일정을 쉽게 확정할 수 있다.

### 4.3. Task & Mission UI
게시판 형태의 UI에서 주차별 할당 미션 목록을 확인할 수 있다. 과제를 제출해야 할 때는 텍스트 입력 및 파일(산출물) 첨부가 가능한 모달 창이 제공된다. 제출 완료 후 관리자가 이를 검토하여 승인하면 각 팀원의 기여도를 나타내는 프로그레스 바(Progress Bar)의 점수와 상태가 갱신된다.

### 4.4. Event Voting & Team Workspace UI
시스템 접속 시 팝업을 통해 이벤트 투표창이 열리며, 미투표자에게는 주기적인 알림 뱃지가 노출된다. 투표가 마감되어 참여자가 확정되면 UI 좌측 또는 상단 탭에 새로운 '팀 워크스페이스' 아이콘이 생성된다. 해당 워크스페이스를 클릭하여 진입하면 목적이 섞이지 않도록 **지시(명령), 대화방, 공지방, 익명방** 등으로 용도별로 깔끔하게 구분된 하위 채널 인터페이스가 제공된다.

## 5. Glossary

| Terms | Description |
| :--- | :--- |
| **관리자(Administrator)** | 조를 생성하고 팀원을 초대하며, 일정을 확정하고 과제를 승인하는 등 시스템 운영의 주요 권한을 가진 사용자이다. |
| **회원(Member)** | 조에 속한 사용자로서 일정을 입력하고, 과제를 업로드하며, 이벤트 투표에 참여할 수 있다. |
| **산출물(Deliverable)** | 미션이나 프로젝트를 통해 생성된 파일, 문서 등을 포괄하여 이르는 말이다. 시스템에 업로드되어 관리자의 승인을 받는다. |
| **블록 처리(Block Processing)** | 캘린더 UI 상에서 본인이 참석 불가능한 시간대를 클릭 혹은 드래그하여 시스템에 불가 시간으로 인식시키는 행위이다. |
| **불가 시간(Unavailable Time)** | 회원이 조모임에 참석할 수 없는 시간대를 의미하며, 캘린더 UI를 통해 입력된다. |
| **미션(Task/Mission)** | 주차별로 회원들에게 할당된 업무 또는 과제로서, 관리자에 의해 공지되며 기여도 점수와 연결된다. |
| **이벤트(Event)** | 전체 회원을 대상으로 투표를 진행하여, 참여 여부에 따라 별도의 소통 그룹으로 분리해야 하는 행사나 단기 모임을 뜻한다. |
| **팀 워크스페이스(Team Workspace)** | 이벤트 투표 결과에 따라 참여 인원만으로 구성된 독립적인 공간이며, 용도별 채널(공지, 대화 등)로 세분화된다. |
