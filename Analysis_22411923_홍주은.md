## 1. Introduction

### 1) Executive Summary
현대의 대학생들은 학업뿐만 아니라 대외활동, 아르바이트 등 매우 바쁜 일상을 보낸다. 특히 대학 생활에서 필수적으로 진행되는 조별 모임의 경우, 개인마다 시간표와 일정이 달라 회의 시간을 정하는 과정부터 극심한 스트레스가 발생하고 있다. 또한, 모임 운영을 위해 개설되는 무분별한 메신저 단체방은 심각한 '정보 피로도'를 유발하며 필수 공지의 가독성을 떨어뜨린다. 종합 모임 관리 프로그램 "Jo:YUl(조율)"은 이러한 대학생 단체 및 조모임의 고질적인 비효율과 소통 피로도를 해결하기 위해 기획되었다. 

### 2) Business Goals
본 시스템은 단순한 연락망을 넘어, 모임의 생성부터 일정 맞춤, 소통, 산출물 관리까지 전 과정을 매끄럽게 '조율'하는 것을 핵심 비즈니스 목적으로 한다. 
첫째, 회원들의 개인 일정을 교차 분석하여 최적의 회의 시간을 자동 추천한다. 둘째, 이벤트나 행사가 있을 때 투표를 통해 참여자를 확정하고 독립적인 그룹 워크스페이스를 생성하여 타겟팅된 일괄 공지를 보낸다. 셋째, 그룹별 산출물과 업무 진행 상황을 한곳에서 통합 관리하여 프리라이더를 방지하고 공정한 참여를 유도한다. 결론적으로 소모적인 연락과 알림 스트레스를 원천적으로 제거하여 협업 환경의 질을 높인다.

### 3) Technical Goals
프로토타입 단계에서는 대규모 데이터베이스 서버 구축 전이므로, 실제 외부 메일 서버(SMTP)를 연동하지 않고 내부 앱(시스템) 알림으로 우선 대체하여 개발한다. 또한 파일 업로드의 경우 텍스트 및 저용량 파일 위주로 지원하며 대용량 영상 파일 업로드는 제한하여 서버 과부하를 방지한다. 인터넷 연결을 기반으로 데이터 및 일정을 동기화하는 구조로 설계한다.

## 3. Domain analysis

본 시스템(Jo:YUl)을 구성하는 주요 클래스 및 도메인 요소는 다음과 같다.
1) **Member**: 시스템을 이용하는 주체. 학번, 이름, 연락처 등의 속성을 가지며, 일반 회원과 관리자, 조장 등으로 나뉜다.
2) **Team/Workspace**: 구성된 조원들의 정보와 그룹 소통 공간을 관리하는 클래스. 이벤트 투표 결과에 따라 자동으로 생성되기도 한다.
3) **Schedule**: 회원들의 가능한 시간과 불가능한 시간 데이터를 담고 있으며, 시스템이 이를 교차 연산하는 데 사용되는 도메인이다.
4) **Task**: 스터디 및 조별 프로젝트 과정에서 발생한 산출물(코드, 파일)과 각 멤버의 업무 진행 상황(상태 바) 데이터를 저장한다.
5) **Event**: 관리자가 주최하는 단기 모임이나 행사 도메인. 참여 투표 상태 및 결과 데이터를 가진다.
6) **System/Server**: 회원 정보, 팀 워크스페이스, 산출물 등을 DB에 동기화하고 일정 교차 추천 알고리즘을 수행하는 중추 역할을 한다.

## 4. User Interface prototype

### 4.1 Register & Log-in UI
* 학번과 비밀번호를 입력할 수 있는 직관적인 로그인 폼 제공.
* 중복 가입 방지를 위해 학번을 ID로 사용하는 회원가입 버튼 배치.

### 4.2 Schedule Recommendation UI (Calendar View)
* 한눈에 들어오는 주간/월간 캘린더 인터페이스 제공.
* 본인이 불가능한 시간대를 마우스 드래그 혹은 클릭만으로 손쉽게 '블록 처리(음영)' 할 수 있도록 구현.
* 조원들의 일정이 합산되어 전원 참석 가능한 시간이 하이라이트 표시됨.

### 4.3 Task Workspace UI (Board)
* 주차별 업무와 담당자 진행 상황을 보여주는 프로그레스 바(Progress Bar) 형태 제공.
* 텍스트 입력 및 파일(산출물) 첨부가 가능한 깔끔한 게시판 형태의 업로드 창.

### 4.4 Event Voting & Notification UI
* 팝업 혹은 상단 알림 바를 통해 새로운 이벤트 등록 사실을 알림.
* 직관적인 [참여] / [불참] 라디오 버튼 투표 창 제공.
* 투표 완료 후 참여자들에게만 열리는 독립적인 그룹 채팅/공지방 형태 UI.

## 5. Glossary

| Terms | Description |
| :--- | :--- |
| **조장/팀장** | 생성된 프로젝트 조의 리더 역할을 수행하며, 팀원 초대 및 일정 확정 권한을 가진다. |
| **산출물** | 스터디나 프로젝트를 통해 생성된 코드, 문서, 발표 자료 등을 포괄하여 이르는 말이다. |
| **블록 처리** | 캘린더 UI 상에서 본인이 참석 불가능한 시간대를 클릭 혹은 드래그하여 시스템에 불가 시간으로 인식시키는 행위이다. |
| **이벤트(Event)** | 전체 회원을 대상으로 투표를 진행하여, 참여 여부에 따라 별도의 소통 및 관리 그룹으로 자동 분리해야 하는 특정 행사나 단기 모임을 뜻한다. |
