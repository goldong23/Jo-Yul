## 1. Introduction

### 1) Executive Summary
현대의 대학생들은 학업뿐만 아니라 대외활동, 아르바이트 등 이른바 '갓생'을 추구하며 매우 바쁜 일상을 보낸다. 특히 대학 생활에서 조별 모임이 차지하는 비중은 상당하며, 개인마다 시간표와 일정이 달라 모두가 모일 수 있는 회의 시간을 정하는 과정부터 극심한 스트레스가 발생하고 있다. 또한, 모임 운영을 위해 개설되는 무분별한 메신저 단체방은 심각한 '정보 피로도'를 유발한다. 응답자의 71%가 단톡방 공지 내용의 가독성이 부족하다고 느꼈으며, 시급한 개선책으로 필수 공지와 단순 홍보 메시지의 분리를 꼽았다. 이러한 비효율과 피로도 문제를 해결하기 위해 종합 모임 관리 프로그램 "Jo:YUl(조율)"을 기획하였다.

### 2) Business Goals
본 시스템은 단순한 연락망을 넘어, 모임의 생성부터 일정 맞춤, 소통, 산출물 관리까지 전 과정을 매끄럽게 '조율'하는 것을 핵심 비즈니스 목적으로 한다. 
첫째, 시스템이 회원들의 개인 일정을 교차 분석하여 최적의 회의 시간을 자동 추천한다. 
둘째, 새로운 이벤트나 행사가 있을 때 이벤트 카드와 참여/불참 투표를 통해 참여자를 분리하고, 참여 의사를 밝힌 인원만 접근하는 이벤트별 워크스페이스를 제공한다.
셋째, 그룹별 산출물과 업무 승인 상태를 한곳에서 통합 관리하여 제출 여부와 진행률을 투명하게 확인하도록 한다.

### 3) Technical Goals
Jo:YUl은 모임 운영에 필요한 회원 인증, 팀 관리, 일정 조율, 과제 제출, 이벤트 투표, 참여자 전용 워크스페이스 기능을 하나의 Android 앱 안에서 제공하는 것을 기술적 목표로 한다. 사용자는 이메일과 비밀번호를 통해 Supabase Auth 기반으로 회원가입 및 로그인을 수행하며, 회원가입 시 닉네임과 권한을 함께 등록한다. 닉네임은 팀 초대와 사용자 식별에 사용되므로 중복되지 않도록 관리한다.
시스템은 팀을 중심 단위로 하여 일정, 과제, 이벤트 데이터를 구분한다. 관리자는 팀을 생성하고 가입된 사용자의 닉네임을 입력하여 팀원을 초대할 수 있으며, 초대된 사용자에게만 해당 팀의 정보가 표시된다. 일정 조율 기능에서는 사용자가 참석 불가능한 시간을 선택하면, 시스템이 팀별 불가 시간을 기준으로 가능한 시간을 계산하여 관리자가 최종 모임 일정을 확정할 수 있도록 한다.
과제 관리 기능에서는 관리자가 팀별 과제를 생성하고, 회원은 Android 파일 선택기를 통해 실제 산출물 파일을 제출한다. 제출 파일은 Supabase Storage에 업로드되며, 앱은 파일명, MIME type, 저장 경로, 제출자 정보를 함께 관리한다. 관리자는 제출된 산출물을 확인하고 승인할 수 있으며, 승인 상태는 팀별 과제 진행률에 반영된다.
이벤트 기능에서는 관리자가 이벤트와 투표 마감 시간을 생성하고, 회원은 참여 또는 불참을 선택한다. 투표 결과는 사용자별로 분리하여 관리하며, 참여로 투표한 사용자에게는 해당 이벤트 전용 워크스페이스가 표시된다. 워크스페이스 메시지는 이벤트 ID별로 분리되어 같은 팀 안의 다른 이벤트와 섞이지 않도록 설계한다.

## 3. Domain analysis

본 시스템(Jo:YUl)을 구성하는 주요 클래스 및 도메인 요소는 다음과 같다.
1) **Member**: 시스템을 이용하는 주체. Supabase 사용자 ID, 이메일, 중복되지 않는 닉네임, 권한(role) 등의 속성을 가지며 일반 회원과 관리자로 나뉜다.
2) **AuthAccount**: Supabase Auth에 저장되는 이메일 기반 인증 계정이다. 로그인 세션과 access token을 통해 앱 기능 접근을 제어한다.
3) **Team**: 관리자에 의해 최초 생성되며 닉네임 기반 초대와 팀원 명단을 관리하는 도메인이다. 팀원 퇴출은 향후 확장 use case로 분리한다.
4) **TeamInvitation**: 관리자가 입력한 닉네임을 기준으로 특정 회원을 팀에 추가하는 초대 정보이다. 초대 전에는 해당 팀이 회원의 팀 목록에 표시되지 않는다.
5) **Schedule**: 회원들의 불가 시간 데이터를 담고 있으며, 시스템이 이를 교차 연산하여 회의 시간을 추천하는 데 사용된다.
6) **Task/Mission**: 관리자가 생성한 팀별 과제와 회원이 제출한 산출물, 승인 상태, 팀별 진행률 계산에 필요한 상태 데이터를 저장한다.
7) **FileStorage**: 사진, 문서, 압축 파일 등 과제 산출물을 Supabase Storage에 저장하고 object path를 제출 정보와 연결한다.
8) **Event**: 관리자가 주최하는 단기 모임이나 행사 도메인. 팀별 참여/불참 투표 상태와 투표 마감 시간을 가진다.
9) **Team Workspace**: 이벤트 투표 결과(참여자)를 기반으로 표시되는 이벤트별 메시지 공간이다. 참여자는 별도 입장 절차 없이 자동 포함되며, 같은 이벤트 안의 메시지만 확인한다.
10) **Server / System**: Supabase Auth 인증, 닉네임 조회, Storage 업로드, 회원탈퇴 Edge Function 호출을 담당한다. 팀, 일정, 과제, 이벤트, 워크스페이스 상태는 현재 Android 구현에서 앱 실행 중 메모리 상태로 관리한다.

## 4. User Interface Flow

Jo:YUl은 모바일 환경에서 사용하는 것을 기준으로 화면 흐름을 구성한다. 하단 내비게이션을 통해 홈, 일정, 미션, 이벤트 화면으로 이동할 수 있으며, 각 화면은 회원가입, 로그인, 팀 초대, 일정 조율, 과제 제출, 이벤트 투표 use case와 연결된다.

### 4.1. Register & Log-in UI
모임 멤버나 관리자가 프로그램을 실행하면 Jo:YUl의 로그인 화면이 가장 먼저 표시된다. 사용자는 이메일과 비밀번호를 입력하여 로그인할 수 있으며, 계정이 없는 경우 회원가입 화면으로 전환할 수 있다. 회원가입 화면에서는 앱에서 사용할 닉네임, 이메일, 비밀번호, 사용자 권한(회원/관리자)을 입력한다. 로그인 성공 후 앱은 사용자 메타데이터의 권한을 기준으로 관리자 기능 노출 여부를 결정한다. 이는 Use Case Diagram에서 `Register member --<<extend>>--> Login` 관계로 표현되는 흐름과 연결된다.

### 4.4. Task & Mission UI

## 5. Glossary

| Terms | Description |
| :--- | :--- |
| **관리자(Administrator)** | 조를 생성하고 팀원을 초대하며, 일정을 확정하고 과제를 승인하는 등 시스템 운영의 주요 권한을 가진 사용자이다. |
| **회원(Member)** | 조에 속한 사용자로서 일정을 입력하고, 과제를 업로드하며, 이벤트 투표에 참여할 수 있다. |
| **산출물(Deliverable)** | 미션이나 프로젝트를 통해 생성된 파일, 문서 등을 포괄하여 이르는 말이다. 시스템에 업로드되어 관리자의 승인을 받는다. |
| **블록 처리(Block Processing)** | 캘린더 UI 상에서 본인이 참석 불가능한 시간대를 클릭 혹은 드래그하여 시스템에 불가 시간으로 인식시키는 행위이다. |
| **불가 시간(Unavailable Time)** | 회원이 조모임에 참석할 수 없는 시간대를 의미하며, 캘린더 UI를 통해 입력된다. |
| **미션(Task/Mission)** | 회원에게 할당된 업무 또는 과제로서, 관리자에 의해 생성되고 제출/승인 상태와 연결된다. |
| **이벤트(Event)** | 팀 안에서 참여/불참 투표를 진행하여 참여자 전용 워크스페이스 표시 기준이 되는 행사나 단기 모임을 뜻한다. |
| **팀 워크스페이스(Team Workspace)** | 이벤트 투표 결과에 따라 참여 인원만으로 표시되는 이벤트별 메시지 공간이다. |
| **Supabase Auth** | 이메일과 비밀번호 기반 회원가입 및 로그인 세션을 관리하는 인증 서비스이다. |
| **Supabase Storage** | 사진, 문서, 압축 파일 등 과제 산출물을 저장하는 파일 저장소이다. |
| **Edge Function** | 회원탈퇴처럼 앱에 직접 넣을 수 없는 서버 권한 작업을 수행하는 Supabase 서버 함수이다. |
| **닉네임(Nickname)** | Jo:YUl에서 사용자를 식별하고 팀 초대에 사용하는 고유 표시 이름이다. 중복을 허용하지 않는다. |
